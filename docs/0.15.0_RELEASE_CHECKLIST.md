# OpenLark 0.15.0 发布清障清单

更新日期：2026-04-05
当前基线：`0.15.0`
目标：在不牺牲公开 API 可解释性、文档可信度和发布纪律的前提下，发布 `0.15.0` 正式版。

## 发布结论门槛

只有当以下四类条件同时满足时，才应发布 `0.15.0`：

- 公开入口、兼容性边界和迁移路径有明确文档。
- README、crate README 和主推示例不再包含过期依赖名、占位链接或错误入口。
- CI 覆盖正式版最低发布信号，而不只是“代码能编过”。
- GitHub Release / crates.io 的 RC 与正式版标记行为正确。

## 发布执行顺序

1. 完成文档、策略和迁移说明。
2. 完成 README / examples 校验自动化并接入 CI。
3. 更新版本文案到 `0.15.0`，补正式版 changelog。
4. 在主干再次执行发布前验证。
5. 创建 `v0.15.0` tag 并确认 GitHub Release 为正式版。
6. 确认 crates.io 根 crate 与子 crate 发布成功。
7. 关闭发布相关 issue，并记录剩余非阻塞风险。
